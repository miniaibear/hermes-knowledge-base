# 知识库架构设计原则

**来源：** [[../原料/知识库架构设计原则.md]] — ElioX 个人经验，2026-04-11

**核心 Takeaway：**
笔记库即代码仓库，原料只读，Claude 是编译器，知识是编译出来的，不是堆出来的。

---

## 架构概览

三层目录（原料/摘要/沉淀）+ 两个元文件（CLAUDE.md + index.md）+ 一条铁律（原料永远不变）

### 三层目录

| 目录 | 定位 | Claude 权限 |
|------|------|------------|
| 原料/ | 只读原始素材 | ❌ 不可修改 |
| 摘要/ | Claude 结构化编译产物 | ✅ 可写 |
| 沉淀/ | 每次 Query 的高质量回答 | ✅ 可写 |

### 两个元文件

- **CLAUDE.md** — 知识库宪法，vault 根目录，每次会话必读。定义 AI 身份、目录规范、工作流、防腐规则
- **index.md** — 全局目录，每篇笔记一行 TLDR。检索时先扫 index 再决定是否深读，节省 Token

### 一条铁律

原料/ 永远不变。知识是编译出来的，不是堆出来的。

---

## 工作流三个关键动作

| 动作 | 触发 | 产出 |
|------|------|------|
| Ingest | 新素材进入原料/ | 摘要/ + 更新 index.md + log.md |
| Query | 用户提问 | 有价值回答存入沉淀/ + 更新 index.md |
| Lint | 定期健康检查 | 发现矛盾标注 ⚠️，不覆盖 |

---

## 防腐规则

1. **[事实] 重要断言必须有来源链接** — 原文/URL/页码
2. **[规则] 新旧冲突时报 diff，不默默覆盖**
3. **[规则] 区分"原文事实"和"推论"** — 分别用 `[事实]` / `[推论]` 前缀标注
