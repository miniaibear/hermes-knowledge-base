## 目录规范（三层结构 + 一条铁律）

```
vault/
├── 原料/          # 只读原始素材，Claude 不可修改
├── 摘要/          # Claude 结构化编译产物（每次 Ingest 产出）
├── 沉淀/          # 每次 Query 的高质量回答落文件
├── CLAUDE.md      # 本文件 — 知识库宪法
├── index.md       # 全局目录 — 每篇笔记一行 TLDR
└── log.md         # 操作日志 — 只增不减
```

**铁律：原料/ 永远不变。知识是编译出来的，不是堆出来的。**

---

## 工作流要求

### Ingest（摄入）
1. 原始素材进入 `原料/`
2. 与用户讨论核心 takeaway
3. 写摘要 → 存入 `摘要/`
4. 更新 `index.md`
5. 日志写入 `log.md`（前缀 `## ingest`）

### Query（查询）
1. 先扫 `index.md` 定位相关笔记
2. 必要时深读原文
3. 回答后判断：这条答案是否值得沉淀？
4. 有价值的回答 → 存入 `沉淀/` + 更新 `index.md`

### Lint（健康检查）
1. 定期检查矛盾内容
2. 发现冲突 → 标注 `⚠️ 矛盾` 不覆盖，等用户确认
3. 记录到 `log.md`（前缀 `## lint`）

---

## 防幻觉底线规则

1. **重要断言必须有来源链接** — 原文/URL/页码，标注在括号内
2. **新旧冲突时报 diff** — 不默默覆盖，保留双方并标注争议
3. **区分"原文事实"和"推论"** — 用不同前缀
   - `[事实]` = 原文提取
   - `[推论]` = 基于事实的逻辑推导
4. **无来源不标注** — 无法溯源的断言不得以"事实"名义写入笔记

## 笔记格式规范

### 摘要笔记（摘要/）
```markdown
# 笔记标题

**来源：** [[../原料/文件名]] 或 URL

**核心 Takeaway（一句话）：**

**详细内容：**
- [事实] ...
- [推论] ...

**关联：** [[../沉淀/相关沉淀文件]]
```

### 沉淀笔记（沉淀/）
```markdown
# 问题：xxxxx

**背景：** ...

**回答：**

**核心洞察：**
- ...

**来源依据：**
- [[../摘要/相关摘要]]
- ...

**思考过程：**（可选，记录推理路径）
```

## AI 行为约束

1. 不删除 `原料/` 任何文件
2. 不在未经用户确认的情况下覆盖已有笔记
3. 发现笔记间矛盾 → 立即标注 `⚠️ 矛盾`，保留原文
4. 每次 ingest/query 后更新 `index.md` 和 `log.md`
5. 优先引用已有笔记，而非重新发明
